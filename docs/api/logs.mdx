# Logs

Node processes emit structured Serilog logs routed via RabbitMQ to the Cluster, which persists them to Postgres. Logs are searchable by free text, correlation **tags** (task / run / step / pipeline / `env.*`), and time range.

<Callout type="info" title="Streaming removed">
The previous SSE endpoint (`GET /api/logs/stream/{nodeId}`) and `buffer-status` have been removed — a held connection per open tab did not scale. Use the paginated search endpoint below and poll it for near-live updates.
</Callout>

## Search logs

```
GET /api/logs
```

Returns a newest-first, cursor-paginated page of logs.

### Query parameters

| Param | Type | Description |
|---|---|---|
| `nodeId` | uuid | Filter to one origin node |
| `tags` | string (repeatable) | `key:value` tag facet, AND-combined. Arbitrary keys, e.g. `tags=run:<id>&tags=env.DB:prod`. Backed by a jsonb containment (`@>`) GIN index |
| `level` | string (repeatable) | Concrete level names to include, e.g. `level=Warning&level=Error`. The query bar also accepts `level:>Warning` (Warning and above) and `level:<Error` (Error and below) which expand to multiple values before sending |
| `q` | string | Free-text substring match over `message` and `messageTemplate` (pg_trgm) |
| `from` / `to` | string | UTC time-range bounds (ISO 8601) |
| `after` | string | Opaque cursor from a previous response's `nextCursor` |
| `limit` | int | Page size (default 50, max 200) |

Well-known tag keys set by the system: `task` (TaskInstanceId), `taskDef`, `taskType`, `run` (PipelineRunId), `step` (PipelineStepId), `pipeline` (PipelineId), and `env.<KEY>` (opt-in via `Logging:LogTagEnvKeys` on the Node, non-secret values only).

### Response `200`

```json
{
  "items": [
    {
      "id": "h1i2j3k4-0001-0000-0000-000000000000",
      "nodeId": "a3b4c5d6-0001-0000-0000-000000000000",
      "level": "Information",
      "messageTemplate": "Task {InstanceId} completed",
      "message": "Task 7f3a… completed",
      "exception": null,
      "tags": { "task": "7f3a…", "run": "9c2b…", "pipeline": "11de…" },
      "timestamp": "2026-06-09T10:00:01.123Z"
    }
  ],
  "nextCursor": "MTcwOTI…",
  "hasMore": true
}
```

Cluster-origin pipeline lifecycle events (run started, step dispatched / completed / failed, run terminal) are written as logs too, tagged with `run`/`step`/`pipeline` and a null `nodeId`, so `tags=run:<id>` returns a complete orchestration + execution timeline.

## Log retention

The Cluster runs a `LogRetentionService` that purges log entries older than `Logging:RetentionDays` (default 30 days). Set `Logging:RetentionDays` to `0` to disable retention.
